# B4 — `activation == "deleted"` docs treated as live

## Summary
`classify_project()` and the invoice-price filter only exclude `activation == "archived"`.
A deleted order that still carries an active status keeps its project marked **Aktiv**
forever. Violates R6 (correct status classification).

## Expected
Deleted docs are dead and excluded from classification and pricing, exactly like archived
docs.

## Fix
- [ ] Use `activation not in ("archived", "deleted")` in `classify_project()` and the invoice-price filter.

## Notes / confirmation
Confirmed against the app (2026-06-10): every EWX view filters `deleted|* = "false"`
server-side — the UI never shows deleted docs, so neither should we. See
[../docs/eventworx API analysis.md](../docs/eventworx%20API%20analysis.md).
