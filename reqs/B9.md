# B9 — Invoice price override not restricted to `finalInvoice`

## Summary
The invoice price override (R8) treats *any* non-cancelled, non-archived invoice as
eligible to set `currentPrice`. Invoices actually carry a `docSubType`
(`finalInvoice` / `invoicecorrection` / `downpayment`) that `_parse_doc_row` does not
capture, so a correction or downpayment invoice can win the override. Found 2026-06-10.

## Impact
- An `applied` correction invoice is eligible today and would win if modified last
  (near-miss observed in **P-1086**).
- A `downpayment` invoice would set `currentPrice` to a partial amount.

## Fix
- [ ] Capture `docSubType` on `Document` in `_parse_doc_row`.
- [ ] Filter the price override to `docSubType == "finalInvoice"` (treat empty as final for backward compatibility).
- [ ] Verify against P-1086 that the correction invoice no longer wins.

## Notes
See "docSubType" in [../docs/eventworx API analysis.md](../docs/eventworx%20API%20analysis.md).
This capture is also a prerequisite for the G5 invoice reminders (R24).
